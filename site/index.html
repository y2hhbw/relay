Relay

Let an AI agent buy its own tokens and use them.

If an agent is going to become a real autonomous entity, it cannot depend on a human topping up its balance forever. Relay gives it the minimum financial layer it needs to top up, pay for upstream AI services, and keep a usable ledger of what it spent.

Why this matters

Tokens are not decoration for an agent. They are the cost of inference, the price of tool use, and the condition for continued operation. An agent that cannot acquire its own balance is still a hosted tool.

What Relay does

Relay gives an agent a deposit address, an API key, a billing layer, and a call ledger. Top up first. Call services next. Charge accurately. Keep the record.

Current MVP

Already implemented

Account bootstrap

Create an account, issue an API key, and assign a dedicated deposit address.

Billing logic

Support fixed-price calls and reserve-then-settle flows in the same gateway.

Ledger and history

Track balances, reservations, settlements, and API call history with queryable records.

Chain listener

Poll USDC transfer events and auto-credit matching deposit addresses.

Relay is the layer between “an agent can think” and “an agent can keep paying to think.”
